EIN

API
Anwendungsprogrammierschnittstelle - eine Rechenschnittstelle, die Interaktionen zwischen mehreren Software-Intermediären definiert. Es definiert die Arten von Anrufen oder Anforderungen, die getätigt werden können, wie sie getätigt werden, die zu verwendenden Datenformate, die zu befolgenden Konventionen usw. Eine API kann vollständig benutzerdefiniert und komponentenspezifisch sein oder auf der Grundlage eines Industriestandards entworfen werden, um die Interoperabilität sicherzustellen.

C.

CDN
Netzwerk für die Bereitstellung von Inhalten - geografisch verteiltes Netzwerk von Proxyservern und deren Rechenzentren. Ziel ist es, eine hohe Verfügbarkeit und Leistung zu erzielen, indem der Dienst räumlich relativ zu den Endbenutzern verteilt wird. CDNs entstanden Ende der 1990er Jahre, um die Leistungsengpässe des Internets zu lindern.
CMS
Content Management System - Computersoftware zur Verwaltung der Erstellung und Änderung digitaler Inhalte. Es gibt viele Web-CMS wie Wordpress und Drupal, die zum Verwalten des Inhalts einer Website verwendet werden. CMS können jedoch zum Verwalten aller Arten von Inhalten verwendet werden. Beispielsweise ist die Video Cloud-Medienbibliothek ein CMS für Medieninhalte und Medienressourcen. Aus diesem Grund wird die Haupt-API, die zur Verwaltung dieser Inhalte verwendet wird, als CMS-API bezeichnet.

D.

STRICH
Dynamisches adaptives Streaming über HTTP - eine adaptive Bitraten-Streaming-Technik, die ein qualitativ hochwertiges Streaming von Medieninhalten über das Internet ermöglicht, die von herkömmlichen HTTP-Webservern bereitgestellt werden. Ähnlich wie bei der HTTP-Live-Streaming (HLS) -Lösung von Apple unterteilt MPEG-DASH den Inhalt in eine Folge kleiner Segmente, die über HTTP bereitgestellt werden.
DRM
Management von Digitalen Rechten - Tools oder technologische Schutzmaßnahmen sind eine Reihe von Zugangskontrolltechnologien zur Einschränkung der Verwendung von proprietärer Hardware und urheberrechtlich geschützten Werken. DRM-Technologien versuchen, die Verwendung, Änderung und Verbreitung von urheberrechtlich geschützten Werken (wie Software und Multimedia-Inhalten) sowie von Systemen auf Geräten zu steuern, die diese Richtlinien durchsetzen.

H.

HLS
HTTP Live Streaming - ein HTTP-basiertes adaptives Bitraten-Streaming-Kommunikationsprotokoll, das von Apple Inc. entwickelt und 2009 veröffentlicht wurde. Die Unterstützung für das Protokoll ist in Mediaplayern, Webbrowsern, Mobilgeräten und Streaming Media-Servern weit verbreitet.
HTTP
Hypertext Transfer Protocol - ein Protokoll auf Anwendungsebene für verteilte, kollaborative Hypermedia-Informationssysteme. HTTP fungiert als Request-Response-Protokoll im Client-Server-Computermodell. Ein Webbrowser kann beispielsweise der Client sein, und eine Anwendung, die auf einem Computer ausgeführt wird, auf dem eine Website gehostet wird, kann der Server sein. Der Client sendet eine HTTP-Anforderungsnachricht an den Server. Der Server, der Ressourcen wie HTML-Dateien und andere Inhalte bereitstellt oder andere Funktionen im Auftrag des Clients ausführt, gibt eine Antwortnachricht an den Client zurück. Die Antwort enthält Informationen zum Abschlussstatus der Anforderung und kann auch angeforderten Inhalt in ihrem Nachrichtentext enthalten.

M.

MP4
MPEG-4 Teil 14 - Ein digitales Multimedia-Container-Format, das am häufigsten zum Speichern von Video und Audio verwendet wird. Es kann jedoch auch zum Speichern anderer Daten wie Untertitel und Standbilder verwendet werden. Im Gegensatz zu HLS und DASH wird MP4 als einzelne Datei und nicht als Segmente geliefert und wird daher häufig in Fällen verwendet, in denen die Zuschauer das Video herunterladen können.

R.

SICH AUSRUHEN
Repräsentativer Staatstransfer - Ein Software-Architekturstil, der eine Reihe von Einschränkungen definiert, die zum Erstellen von Webdiensten verwendet werden sollen. Webdienste, die dem REST-Architekturstil entsprechen und als RESTful Web Services bezeichnet werden, bieten Interoperabilität zwischen Computersystemen im Internet. Mit RESTful-Webdiensten können die anfordernden Systeme auf Textdarstellungen von Webressourcen zugreifen und diese bearbeiten, indem sie einen einheitlichen und vordefinierten Satz zustandsloser Vorgänge verwenden. Brightcove verwendet RESTful-APIs, um auf verschiedene Teile der Plattform zuzugreifen.
RTMP
Echtzeit-Messaging-Protokoll - Zunächst ein von Macromedia entwickeltes proprietäres Protokoll für das Streaming von Audio, Video und Daten über das Internet zwischen einem Flash-Player und einem Server. Macromedia gehört jetzt Adobe, das eine unvollständige Version der Spezifikation des Protokolls für die öffentliche Verwendung freigegeben hat.
