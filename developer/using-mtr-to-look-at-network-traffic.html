Einführung

MTR ist ein beliebtes Tool zur Diagnose von Netzwerkproblemen. Es kombiniert die Funktionalität der ping und traceroute Befehle. Das Einreichen von MTR-Berichten kann dem Brightcove-Support in einigen Fällen helfen, Probleme zu beheben.

Sehen So führen Sie eine MTR aus Informationen zur Installation von MTR.

Beachten Sie, dass Homebrew jetzt MTR in ein Nicht-Pfad-Verzeichnis installiert (zumindest auf dem Mac). Um MTR ausführen zu können, müssen Sie im Terminal Folgendes tun: 
cd /usr/local/Cellar/mtr/
# to get the version number - in this case 0.94 - and cd into that directory's /sbin directory
ls
cd 0.94/sbin
# now copy mtr to the path
sudo cp mtr /usr/local/bin

Jetzt kannst du laufen sudo mtr (mit sudo erforderlich)

Die folgenden Schritte führen Sie durch die grundlegende Installation und Verwendung von Wireshark.

Verwenden von MTR mit Live

Die Verwendung von MTR mit Live-Streams ähnelt der Analyse von VOD, unterscheidet sich jedoch ein wenig, da MTR ICMP-Anforderungspakete sendet und die meisten Internetgeräte ICMP-basierte Trace-Anfragen weniger priorisieren und ICMP-Datenverkehr absichtlich drosseln, was die Illusion eines Paketverlusts erzeugen kann.

Darüber hinaus müssen Sie MTR sowohl auf der RTMP-URL als auch auf der Wiedergabe-URL ausführen, da sowohl bei der Eingabe als auch bei der Ausgabe Probleme auftreten können.

Um Verbindungsprobleme ordnungsgemäß zu beheben, muss eine TCP-basierte Ablaufverfolgung ausgeführt werden. Unten sehen Sie ein Beispiel, das auf der RMTP-URL für Mac-Benutzer ausgeführt wird.

sudo mtr -n -T -c 200 1a244fa8f08a464e869695dd01c890d6.sep.bcovlive.io --report
