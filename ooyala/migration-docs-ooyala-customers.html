Migrationsdokumente für Ooyala-Kunden

Die hier aufgeführten Themen beziehen sich auf die Konnektoren und Adapter, die Brightcove entwickelt hat, um Ooyala-Kunden beim Übergang zur Video Cloud zu unterstützen.

Hinweis: Diese Themen gelten nur an Ooyala-Kunden, die zu Video Cloud migriert wurden.

Schauen Sie regelmäßig hierher, da Brightcove an zusätzlichen Tools arbeitet, die bei der Migration von Ooyala zu Video Cloud helfen.

  • Verwenden der Ooyala-Player zum Bereitstellen von Video-Cloud-Inhalten
  • Hinzufügen des Brightcove-DNS-Servers
  • Backlot / Video Cloud CMS Connector
  • Backlot und Video Cloud: Video- und Wiedergabelistenfelder
  • Backlot-API-Referenz (Brightcove-Version)
  • Migration zur Video Cloud: Aufnahmeadapter
    • Ooyala-Video Cloud Verschluckungsadapter: Einnahme mit OIS 2.5
    • Ooyala-Video Cloud Verschluckungsadapter: Manifest-Dateiformate
    • Ooyala-Video Cloud Verschluckungsadapter: Eingabedateiformate
    • Ooyala-Video Cloud Verschluckungsadapter: Einnahme mit Aspera
    • Ooyala-Video Cloud Verschluckungsadapter: Einnahme mit SFTP
    • Ooyala-Video Cloud Verschluckungsadapter: Einnahme mit MRSS
    • Ooyala-Video Cloud Verschluckungsadapter: Verwenden von MRSS zum Erstellen von Remote-Assets
    • Ooyala-Video Cloud Verschluckungsadapter: CSV-Manifestdateien
    • Ooyala-Video Cloud Verschluckungsadapter: XML-Manifestdateien
  • Migration zur Video Cloud: IQ API und Video Cloud Analytics API
  • Authentifizierung für Video Cloud Platform-APIs
