Ooyala-Video Cloud Verschluckungsadapter: CSV-Manifestdateien

Sie können weiterhin FTP oder Aspera mit einem CSV-Manifest verwenden, um Ihre Videos in Video Cloud aufzunehmen. In diesem Thema wird die Struktur der CSV erläutert.
Hinweis: Dieses Thema gilt nur an Ooyala-Kunden, die zu Video Cloud migriert wurden.

Einführung

Beim Aufnehmen von Videos können Sie CSV-Manifestdateien verwenden, um zusätzliche Metadaten (einschließlich Videotitel, Beschreibung, Flugzeiten, Beschriftungen und benutzerdefinierte Metadaten) sowie Miniatur- und Untertiteldateien anzugeben. Sie können CSV-Manifestdateien verwenden, wenn Sie Inhalte mit FTP oder Aspera aufnehmen.

Einschränkungen

Hinweis: Dieses Thema gilt für OIS v2.5.

Das CSV-Format der Metadatendatei unterstützt weniger Funktionen als das XML-Format. Im CSV-Format können Sie Miniaturansichten (Vorschaubilder) und benutzerdefinierte Metadaten angeben, die einem Video zugeordnet werden sollen. Das XML-Format unterstützt diese Funktionen sowie die Zuordnung der Namen vorhandener definierter Anzeigensätze.

Umgang mit Metadaten

Wenn Sie einschließen metadata Videos, die Sie aufnehmen, werden auf zwei Arten Video Video-Videofeldern zugeordnet:

  1. Standardmäßig wird jedes metadata Schlüssel/Wert-Paar als Tag in der Form: meta::key::value. Beispielsweise, <ooyala:metadata name="category">cooking</ooyala:metadata> würde das Tag produzieren meta::category::cooking
  2. Wenn Sie lieber verwenden möchten custom_fields Um Metadaten zu speichern, kann der Brightcove-Support eine Einstellung in Ihrem Konto ändern, um dies zu tun. Beachten Sie Folgendes:
    • Sie müssen benutzerdefinierte Felder mit Namen erstellen, die Ihren Metadatenschlüsseln entsprechen.
    • Die Anzahl der benutzerdefinierten Felder ist standardmäßig auf 10 begrenzt. Wenn Sie also mehr als das benötigen, um Ihre Metadaten aufzunehmen, müssen Sie zusätzliche benutzerdefinierte Felder vom Support anfordern.
    • Wenn in Ihrem Konto kein benutzerdefiniertes Feld für jedes Metadatenelement vorhanden ist, schlägt der gesamte Metadatenvorgang fehl.

Spaltenüberschriften des CSV-Manifests

Die folgenden Felder sind die erste Zeile (Spaltenüberschriften) Ihrer CSV-Datei, wobei die tatsächlichen Datenfelder in den nachfolgenden Zeilen folgen (eine Zeile pro Video).

Hinweis: Sie können diese Spalten in beliebiger Reihenfolge angeben und nicht benötigte Spalten weglassen, solange die Daten in den Datenzeilen genau mit den Überschriften übereinstimmen.

Erstellen einer CSV-Manifestdatei

Beachten Sie beim Generieren einer CSV-Metadatendatei Folgendes:

  • Wenn ein Textwert in einem Datenfeld ein Komma (,) hat, müssen Sie das Feld in doppelte Anführungszeichen (") setzen.
  • Wenn ein Textwert in einem Datenfeld ein doppeltes Anführungszeichen („) enthält, müssen Sie das Feld in doppelte Anführungszeichen setzen und dem doppelten Anführungszeichen im Textfeld mit einem zweiten Anführungszeichen entgehen (z. B. „Sie sagte: „“ Habt einen schönen Tag. „"“)

Wenn Sie Ihre CSV-Metadatendatei aus Excel erstellen, wird das Escapezeichen automatisch verarbeitet.

So erstellen Sie eine CSV-Metadatendatei:

  1. Laden Sie ein oder mehrere Videos hoch.
  2. Öffnen Sie ein Tabellenkalkulationsprogramm wie Microsoft Excel.
  3. Fügen Sie die Spaltenüberschriften zur ersten Zeile hinzu.
  4. Fügen Sie für jedes Video eine Zeile hinzu.
  5. Speichern Sie die Datei als CSV. Zum Beispiel könnten Sie auswählen Speichern als von dem Datei Speisekarte. Wenn Sie dazu aufgefordert werden, wählen Sie den durch Kommas getrennten Dateityp aus.
  6. Laden Sie mit der Upload-Methode Ihrer Wahl die CSV-Datei hoch, während Sie die Videos hochgeladen haben. Siehe entweder FTP oder Aspera.
