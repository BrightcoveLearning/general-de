Ooyala-Video Cloud Verschluckungsadapter: XML-Manifestdateien

Sie können weiterhin FTP, Aspera oder MRSS mit einem XML-Manifest verwenden, um Ihre Videos in Video Cloud aufzunehmen. In diesem Thema wird die Struktur des XML erläutert.
Hinweis: Dieses Thema gilt nur an Ooyala-Kunden, die zu Video Cloud migriert wurden.

Einführung

Beim Aufnehmen von Videos können Sie XML-Manifestdateien verwenden, um zusätzliche Metadaten (einschließlich Videotitel, Beschreibung, Flugzeiten, Beschriftungen und benutzerdefinierte Metadaten) sowie Miniatur- und Untertiteldateien anzugeben. Sie können XML-Manifestdateien verwenden, wenn Sie Inhalte mit FTP, Aspera oder MRSS aufnehmen.

Hinweis: Dieses Thema gilt für OIS v2.5.

Umgang mit Metadaten

Wenn Sie einschließen metadata Videos, die Sie aufnehmen, werden auf zwei Arten Video Video-Videofeldern zugeordnet:

  1. Standardmäßig wird jedes metadata Schlüssel/Wert-Paar als Tag in der Form: meta::key::value. Beispielsweise, <ooyala:metadata name="category">cooking</ooyala:metadata> würde das Tag produzieren meta::category::cooking
  2. Wenn Sie lieber verwenden möchten custom_fields Um Metadaten zu speichern, kann der Brightcove-Support eine Einstellung in Ihrem Konto ändern, um dies zu tun. Beachten Sie Folgendes:
    • Sie müssen benutzerdefinierte Felder mit Namen erstellen, die Ihren Metadatenschlüsseln entsprechen.
    • Die Anzahl der benutzerdefinierten Felder ist standardmäßig auf 10 begrenzt. Wenn Sie also mehr als das benötigen, um Ihre Metadaten aufzunehmen, müssen Sie zusätzliche benutzerdefinierte Felder vom Support anfordern.
    • Wenn in Ihrem Konto kein benutzerdefiniertes Feld für jedes Metadatenelement vorhanden ist, schlägt der gesamte Metadatenvorgang fehl.

XML-Manifest-Elemente

Sie können die folgenden Elemente in der XML-Manifestdatei verwenden.

XML-Manifest-Elemente
XML-Tag Beschreibung Erforderlich / Optional
<Kanal> Enthält eine oder mehrere <item> -Elemente. erforderlich
<item> Stellt ein einzelnes Video dar, das aufgenommen werden soll. Sie können mehrere Videos angeben (<item> -Tags) in einer XML-Manifestdatei. Enthält die verbleibenden Tags in dieser Tabelle. erforderlich
<Medieninhalt> Gibt ein Video an, das aufgenommen werden soll. Einer der folgenden:
  • Dateiname eines Videos, das Sie hochgeladen haben.
  • Zur Einnahme über MRSS (siehe Aufnahme von einem remote gehosteten MRSS-Feed), die URL, unter der sich das Video befindet. Die URL kann enthalten http:// oder https://.
Verwenden Sie für ein Remote-Asset <ooyala:remoteasset> stattdessen.		 erforderlich (außer für Remote-Assets)
<Medien: Titel> Name oder Titel des Videos. fakultativ
<Medien: Miniaturansicht> Gibt eine Miniaturansicht an (Vorschaubild). Einer der folgenden:
  • Dateiname eines Thumbnails, das Sie hochgeladen haben.
  • URL, unter der sich die Miniaturansicht befindet. Die URL kann enthalten http:// oder https://.
 fakultativ
<Medien:Untertitel> Gibt eine Untertiteldatei an (unterstützt für DFXP-, WebVTT- und SRT-Dateien). Sehen Aufnahme von Untertiteldateien. Attribute:
  • href (erforderlich) - Eine der folgenden:
    • Dateiname der Untertiteldatei, die Sie hochgeladen haben. Beispiel: caption-1.dfxp
    • URL, unter der sich die Untertiteldatei befindet. Die URL kann enthalten http:// oder https://.
    lang (optional) - Sprache der Untertiteldatei. Zwei-Buchstaben-Code. Sehen Unterstützte Untertitel.
  • FrameErate (optional) - Bildrate der Untertiteldatei. Einheiten sind in Frames Per Second (FPS) angegeben. Standard ist 30. Ältere Untertiteldateien sind möglicherweise 24.
Hinweis: Falls angegeben, liegt es in Ihrer Verantwortung, gültige Werte für lang und FrameErate. Gültige Werte werden zur DFXP-Datei hinzugefügt. Nicht konforme Werte werden ignoriert. In beiden Fällen wird die Untertiteldatei zu Backlot hinzugefügt und mit dem Asset verknüpft.
 fakultativ
<Medien: Beschreibung> Beschreibung des Videos. fakultativ
<Verknüpfung> Permanente URL, unter der Sie das Video einbetten. Zuordnung zu den erstellten Assets hosted_at eigentum. fakultativ
<dcterms: valid> Flugzeiten, die die Start- und Endzeiten darstellen, in denen das Asset gespielt werden kann. Beispiel: 
  <dcTerms:valid>start=2011-06-28T 15:15:00-07:00; end=2022-12-01T 16:00:00-07:00; scheme=W3c-DTF </dcterms:valid >
 fakultativ
<ooyala: Etiketten> Ein oder mehrere durch Kommas getrennte Bezeichnungen. Beispiel: /sport, /sports/joggen. Hintergrundinformationen finden Sie unter Etiketten. fakultativ
<media:schlüsselwords> Eine andere Möglichkeit, Beschriftungen zu definieren (ohne Schrägstrich). Der durch Kommas getrennte Inhalt wird dem Video als Tags hinzugefügt. Ein neues Tag wird nur erstellt, wenn noch kein gleichnamiges Tag vorhanden ist.

Zum Beispiel für Folgendes <media:schlüsselwords> tag, die etiketten beschrift1 und beschrift2 erstellt und dem Video als Tags hinzugefügt. Ein Label namens Elternlabel würde auch erstellt werden, mit dem Label untergeordnetes Label darin verschachtelt.

  <media:schlüsselwords>label1, label2, parentlabel/childlabel</media:keywords >
Hinweis: Die Inhalte von <media:schlüsselwords> werden ignoriert, wenn die <ooyala: Etiketten> -Tag ist in der XML-Manifestdatei oder MRSS enthalten.
 fakultativ
<ooyala:embedcode> Video Cloud Video ID. optional (erforderlich, wenn Sie ein Video aktualisieren, anstatt ein neues hinzuzufügen)
<ooyala:profileguid> ID des Verarbeitungsprofils, bei dem es sich um eine Gruppe von Codierungen handelt, die die Formate definieren, in die ein Mastervideo während der Transcodierung konvertiert werden muss. Weitere Informationen erhalten Sie von Ihrem Ooyala-Supportmitarbeiter. fakultativ
<ooyala:metadaten> Benutzerdefinierte Metadaten für das Video. Sie können für jeden Metadatentyp ein Element festlegen. Siehe die Umgang mit Metadaten Abschnitt für Details darüber, wie metadata wird Videos als spezielle Tags hinzugefügt. Beispiel: <ooyala:metadata name="category">cooking</ooyala:metadata> fakultativ
<ooyala:remoteasset> Gibt ein Remote-Asset an, das aufgenommen werden soll. Ein Remote-Asset ist ein Video, das an einem anderen Ort als Backlot gehostet wird. Geben Sie die folgenden Attribute an:
  • DurationInms: Mediendauer (in Millisekunden)
  • url:: URL des Remote-Assets. Die URL kann enthalten http:// oder https://.
Diese Einstellung entspricht dem Wert von hosted_at -Eigenschaft für ein Remote-Asset. Beispiel: <ooyala:remoteasset durationInms='5000' url=“ http://your_website/your_remote_assets.mp4 "/>
Hinweis: Nachdem ein Remote-Asset erstellt wurde, kann sich seine Weitergabe an die verschiedenen CDNs um 60 Sekunden oder mehr verzögern. Wenn Sie ein Remote-Asset zu früh nach seiner Erstellung anfordern, werden die Ergebnisse von den CDNs zwischengespeichert. Das Löschen kann einige Minuten dauern. Best Practice: Nach der Erstellung 30 oder 60 Sekunden warten, Abfrage mit dem Backlot-API [GET] /v2/assets/asset_id route, und fahren Sie nach dem Abrufen des Einbettungscodes des Remote-Assets (Content-ID oder Asset-ID) mit der Einbettung des Assets fort.
 erforderlich (nur Remote-Assets)
<ooyala:profileguid> Gibt ein Video Cloud-Aufnahmeprofil an, das für die Verarbeitung des Videos verwendet werden soll. Wenn Sie das Standardprofil des Kontos nicht verwenden möchten, müssen Sie das Profil angeben, das Sie stattdessen verwenden möchten. Sehen Verwalten von Aufnahmeprofilen. fakultativ

Schritt für Schritt

Erstellen einer XML-Manifestdatei

So erstellen Sie eine XML-Metadatendatei:

  1. Laden Sie Ihre Videos und andere Inhalte hoch (Miniaturansichten, Untertiteldateien usw.).
  2. Öffnen Sie einen Texteditor, vorzugsweise einen, der XML validiert.
  3. Fügen Sie die erforderlichen XML-Header hinzu. Beispielsweise: 
      <? xml version="1.0" encoding="utf-8"? > <rss version="2.0" xmlns:media=“ http://search.yahoo.com/mrss/ "xmlns:dcterms=“ http://purl.org/dc/terms/ "xmlns:fh=“ http://purl.org/syndication/history/1.0 "xmlns:ooyala=“ http://www.ooyala.com/mrss/ „>
  4. Erstellen Sie ein Kanal -Behälter.
  5. Erstelle ein Artikel Eintrag für jedes Video, das die gewünschten Elemente aus der obigen Tabelle enthält.

    Im folgenden Beispiel gibt das XML Metadaten für zwei Videos an. Für das erste Video befinden sich die Datei und die Miniaturansichten auf einem Upload-Server und werden von diesem Server abgerufen. Für das zweite Video wurden die Datei und das Miniaturbild bereits auf einen Ooyala-Server hochgeladen. Das zweite Video hat auch Flugzeiten.

      <? xml version="1.0" encoding="utf-8"? > <rss version="2.0" xmlns:media=“ http://search.yahoo.com/mrss/ "xmlns:dcterms=“ http://purl.org/dc/terms/ "xmlns:fh=“ http://purl.org/syndication/history/1.0 "xmlns:ooyala=“ http://www.ooyala.com/mrss/ „> <kanal> <item> <media:content url=“ http://mysite.com/upload/lacrosse_70.mov "/> <media:thumbnail url=“ http://mysite.com/upload/lacrosse_70_previewimage.jpg "/> <media:title>Sport, die wirklich existieren: Lacrosse</media:title > <media:Beschreibung>Meine Beschreibung meines Videos</media:description > <ooyala:labels>/sports/lacrosse,/ hobbyes/lacrosse</ooyala:labels > <link> http://mysite.com/videos/sports/real_sports.html</link > <ooyala:metadatenname=“ Video_ID">70</ooyala:metadaten > <ooyala:metadatenname="season_number">1</ooyala:metadaten > </item > <item> <media:content url=“ curling_71.mov "/> <media:thumbnail url=“ curling_71_previewimage.jpg "/> <Medien:Titel>Sport, die wirklich existieren: Curling</media:title > <media:Description>Meine Beschreibung meines Videos</media:description > <ooyala:labels>/sports/curling,/ hobbyes/curling</ooyala:labels > <link> http://mysite.com/videos/sports/real_sports.html</link > <ooyala:metadatenname=“ Video_ID">71</ooyala:metadaten > <ooyala:metadatenname = "season_number">1</ooyala:metadata > <! — Flugzeiten festlegen —> <dcTerms:Valid>Start=2011-06-28T 15:15:00-07:00; end=2022-12-01T 16:00:00-07:00; scheme=W3C-DTF </dcterms:valid > </item > </channel > </rss >
  6. Speichern Sie die Datei als XML. Stellen Sie sicher, dass Sie einen eindeutigen Namen angeben. Wenn Sie zwei Dateien mit demselben Namen angeben und eine hochladen, während die andere verarbeitet wird, wird die zweite möglicherweise ignoriert.
  7. Laden Sie mit der Upload-Methode Ihrer Wahl die XML-Manifestdatei hoch, während Sie die Videos hochgeladen haben. Siehe entweder FTP oder Aspera.
