Hinweis: Dieses Thema gilt nur an Ooyala-Kunden, die zu Video Cloud migriert wurden.

Neue Video Cloud-Optionen

Video Cloud bietet drei Standardoptionen für die Aufnahme von Videos:

  • Das Upload-Modul in Studio
  • Pull-basierte Aufnahme mit der Dynamic Ingest API
  • Hochladen der Quelldatei mithilfe der Dynamic Ingest-API

Oder verwenden Sie Ooyala-Aufnahmemethoden

Als Video Cloud-Kunde können Sie eine der oben aufgeführten Optionen verwenden. Brightcove hat jedoch auch Adapter für Ooyala-Aufnahmesysteme entwickelt, um die Videos an das Video Cloud-Aufnahmesystem weiterzuleiten, sodass Sie die weiterhin verwenden können Ooyala Ingestion Service (OIS) So senden Sie Ihre Videos an Video Cloud mit:

  • FTPS
  • Aspera
  • Remote-Hosting von MRSS-Feeds
  • Die Backlot-API

Alle vorhandenen Manifestoptionen funktionieren weiterhin. Verwenden Sie weiterhin Ihre Authentifizierungsdaten für Ooyala, und der Adapter übersetzt diese automatisch in Anmeldeinformationen für die erforderlichen Video Cloud-APIs:

API-basierte Aufnahme
Wenn Ihr Ingestion-Workflow das Aufnehmen neuer Inhalte direkt über die Ingestion-API umfasst: Wenn Ihr Client-Datenverkehr an Video Cloud geleitet wird, verwenden Sie den vorhandenen Ooyala-API-Schlüssel und das Geheimnis, um die API-Aufrufe an die aktuellen Ooyala-API-Endpunkte durchzuführen.
FTP / Aspera-basierte Einnahme
Stellen Sie sicher, dass Sie das Kennwort für das Benutzerkonto "Nur hochladen" auf etwas anderes zurücksetzen können als in Backlot. Wenn Ihr Konto "Nur hochladen" in Backlot sowohl Video als auch Metadaten unterstützt, wird dies auch vom Konto "Nur hochladen" der Video Cloud ausgeführt. Sie können weiterhin eine Verbindung zu denselben FTP / Aspera-Endpunkten herstellen und Ihre neuen Video Cloud-Anmeldeinformationen für das Benutzerkonto "nur hochladen" zum Hochladen von Dateien verwenden. Sobald Dateien hochgeladen wurden und aus dem Upload-Ordner verschwunden sind, melden Sie sich bei Video Cloud an und stellen Sie sicher, dass Ihr Video hinzugefügt wurde. Wenn Sie XML- oder CSV-Metadatendateien zusammen mit Video hochladen und auf ein bestimmtes Aufnahmeprofil abzielen möchten, müssen Sie den Namen des Video Cloud-Profils im Feld XML oder CSV unter profileguid element angeben. Sehen Verwalten von Aufnahmeprofilen.
MRSS-basierte Einnahme
Bitte teilen Sie Ihren MRSS-Feed Ihrem Account-Team mit, damit dieser ihn aktivieren kann. Da wir nicht überprüfen können, welche Videos aus dem MRSS-Feed zuvor in Backlot aufgenommen wurden, stellen Sie sicher, dass nur neue Videos im MRSS-Feed verfügbar sind. Wenn alte Videos im Feed verfügbar sind, werden sie ebenfalls aufgenommen, was zu doppelten Inhalten in der Video Cloud führt. Sobald der MRSS-Feed vorhanden ist, können Sie sich bei Video Cloud anmelden, um zu überprüfen, ob Inhalte aufgenommen werden.
Aufnahme über Brightcove Video Cloud Studio
Ein wichtiger Unterschied zu Backlot ist das Aufnahmeprofil, das Sie für die Aufnahme Ihrer Inhalte auswählen. Wählen Sie in Video Cloud ein Aufnahmeprofil aus, das Ihren Anforderungen am besten entspricht. In finden Sie hilfreiche Informationen zu Video Cloud-Aufnahmeprofilen Verwalten von Aufnahmeprofilen.
