Ooyala-Video Cloud Ingestion Adapter: Unterstützte Ingest-Formate

In diesem Thema wird die Verwendung von CSV-Manifesten zum Aufnehmen von Dateien über das Programm erläutert Ooyala-Video Cloud-Aufnahmeanschluss
Hinweis: Dieses Thema gilt nur an Ooyala-Kunden, die zu Video Cloud migriert wurden.

Aufnahmeformate

In der folgenden Tabelle sind die unterstützten Aufnahmeformate aufgeführt.

  • Empfohlen:: H.264
  • Andere unterstützte Formate: siehe Unterstützte Video- und Audioformate, Codecs und Container
Audio-Codecs

Dateinamen

Während des Aufnahmeprozesses muss der Name der Quelldatei als Teil einer API-Anforderungs-URL übergeben werden. Aus diesem Grund muss der Name URI-kompatibel sein. Vermeiden Sie Folgendes in den Quelldateinamen für alle Videos und Assets:

  • Leerzeichen: statt my video.mov verwenden my-video.mov oder my_video.mov
  • Spezielle Charaktere : tun nicht Verwenden Sie die folgenden Zeichen in Quelldateinamen: "!", "@", "#", "$", "%", "^", "&", "*", "(", ")", "?", "/", "\", "{", "}", "[", "]"

Unterstützte Dateiformate

Videodateien

Schon seit Brightcove unterstützt so viele Videoeingangstypen, dass es hilfreicher ist, über die Formate zu sprechen, die wir nicht unterstützen. Im Moment gibt es nur wenige. Die folgende Liste zeigt die Formate nicht unterstützt für Video Cloud.

  • Apple Intermediate
  • HDV 720p60
  • Go2Meeting3 (G2M3)
  • Avid Meridien Uncompressed
  • Windows Media Audio Lossless
  • CineForm- teilweise unterstützt; Es gibt Varianten, die von der Software stammen können, die zum Codieren der Quelle verwendet wird, oder vom Betriebssystem, die die Quelle von Video Cloud unbrauchbar machen. Wir versuchen immer noch, die genauen Faktoren zu identifizieren, die Probleme verursachen.
  • SWF

Sonderfälle

  • Notiz: AVdh (DNxHQ) geht auch nicht für 422 10-Bit- und 444 12-Bit-Quellen (nur) - Dies ist ein Fehler, und dieser Hinweis wird entfernt, wenn er behoben ist.
  • Mit KMS verschlüsselte Inhalte werden unterstützt, erfordern jedoch, dass Sie Ihre Objekt-URLs vorsignieren, unter anderem wie hier beschrieben: https://aws.amazon.com/blogs/developer/generating-amazon-s3-pre-signed-urls-with-sse-kms-part-2/.

    Zu implementierende Schritte:

    1. Erstellen Sie einen von KMS verwalteten verschlüsselten Bucket
    2. Fügen Sie die Video Cloud / Zencoder-Aufnahmerichtlinie hinzu
    3. Erstellen Sie mit dem KMS-Schlüssel eine vorsignierte URL
    4. Verwenden Sie diese URL, wenn Sie das Video aufnehmen

Abgesehen von den oben aufgeführten glauben wir, dass wir so ziemlich alle bekannten Videoformate unterstützen. Bei Problemen mit einem anderen Videoformat bitte Kontakt Brightcove Unterstützung und lass auch Brightcove Lerndienste kennt.

Audiodateien

Die meisten Formate werden unterstützt, außer wenn sie über 8 Audiokanäle empfangen. (Und das Audio muss entweder als einzelne Mehrkanalspur oder als mehrere Monospuren konfiguriert werden.)

Hinweis: Wenn Ihre Quellen für die Aufnahme von Video Cloud über Mehrkanal-Audio verfügen, benötigen Sie Unterstützung beim Erstellen benutzerdefinierter Audio-Formatvarianten, um die Kanäle korrekt zuzuordnen.

Untertiteldateien

Aufnahme von Untertiteldateien. Beachten Sie, dass HTML5-Webplayer wie die Brightcove- und Ooyala-Player WebVTT-Untertitel benötigen. Wenn du aufnehmen DFXP- oder SRT-Untertitel werden automatisch in WebVTT konvertiert. Für Remote-Asset-Untertitel haben Sie Muss Verwenden Sie das WebVTT-Format.

jpg, jpeg, png und gif

Empfohlene Vorgehensweise

Die folgenden Empfehlungen stellen sicher, dass Ihre Quelldateien bereit sind, die bestmöglichen Ergebnisse zu erzielen.

Progressiver Scan, de-interlaced

Das Quellvideo sollte vor dem Hochladen in die Video Cloud progressiv gescannt werden (progressiv, nicht interlaced, de interlaced). Obwohl Ooyala den Inhalt bei der Verarbeitung des Quellvideos entschachteln kann, empfehlen wir, dies vor dem Aufnehmen zu tun, wenn Sie über die Ressourcen verfügen. Auf diese Weise können Sie die Qualität Ihrer Videoquellen in der Phase der Qualitätskontrolle Ihres redaktionellen Workflows bestätigen.
