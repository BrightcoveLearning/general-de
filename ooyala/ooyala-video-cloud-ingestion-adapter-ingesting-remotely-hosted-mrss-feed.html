Ooyala-Video Cloud Ingestion Adapter: Aufnahme aus einem remote gehosteten MRSS-Feed

Sie können weiterhin Videos und Assets mithilfe von MRSS-Feeds über die Video Cloud in die Video Cloud aufnehmen Aufnahmeadapter.
Hinweis: Dieses Thema gilt nur an Ooyala-Kunden, die zu Video Cloud migriert wurden.

Einführung

Sie können Media RSS (MRSS) verwenden, damit Ooyala Ihre Inhalte aus einem von Ihnen veröffentlichten MRSS-Feed von Ihrem eigenen System abruft. Video Cloud ruft die Metadaten aus dem MRSS-Feed ab und erstellt entweder ein Remote-Asset oder Downloads (von wo sie gehostet werden) und verarbeitet die Videos und alle zugehörigen Dateien (Miniaturansichten, Untertiteldateien usw.).

Hinweis: Dieses Thema gilt für OIS v2.5.

Übersicht über MRSS Ingest

Über MRSS

Media RSS (MRSS) ist eine Erweiterung von RSS, mit der Sie Multimediadateien in RSS-Feeds syndizieren können. Technische Details finden Sie in der Medien-RSS-Spezifikation.

MRSS einrichten

Wenden Sie sich an Ihren Account Manager und geben Sie an, dass Sie MRSS verwenden möchten. Für jedes Video, das Sie aufnehmen möchten, müssen Sie Ooyala gültige Inhalts-URLs bereitstellen. URLs können enthalten http:// oder https://.

MRSS verwenden

Um MRSS zu verwenden, hosten Sie Metadaten und / oder Videos und andere Dateien (Miniaturansichten, Untertitel) auf einem Webserver und erstellen eine XML-Manifestdatei, die auf diesen Inhalt verweist.

Verwenden von MRSS zum Abrufen nur von Metadaten

  1. Erstellen Sie eine XML-Manifestdatei wie in beschrieben XML-Manifestdatei. Sie müssen die verwenden ooyala:remoteasset -Tag für die reine Metadatenaufnahme.
  2. Hosten Sie die XML-Manifestdatei an einem der von Ihnen angegebenen MRSS-Speicherorte. Stellen Sie sicher, dass der Dateiname und der Pfad genau mit einem der Pfade übereinstimmen, die Sie Ooyala zur Verfügung gestellt haben.

Verwenden von MRSS zum Abrufen von Videos, Miniaturansichten und Untertiteldateien

  1. Hosten Sie eine oder mehrere Dateien (Videos, Miniaturansichten und Untertiteldateien) auf einem Webserver.
  2. Erstellen Sie eine XML-Manifestdatei (siehe XML-Manifestdatei). Stellen Sie sicher, dass die Einträge in der Datei auf die Dateien auf Ihrem Server verweisen. Insbesondere der Wert der <Medieninhalt> element's href Attribut muss eine HTTP-URL zu der Datei auf Ihrem Server sein, wie im folgenden Beispiel: 
    <media: content url = "http://mysite.com/upload/lacrosse_70.mov" />
  3. Hosten Sie die XML-Manifestdatei an einem der von Ihnen angegebenen MRSS-Speicherorte. Stellen Sie sicher, dass der Dateiname und der Pfad genau mit einem der Pfade übereinstimmen, die Sie Ooyala zur Verfügung gestellt haben.

    Hinweis: Leider gilt das Konzept der "Client-Protokollierung" nicht für die MRSS-Feed-basierte Verarbeitung. Der beste Weg, um zu überprüfen, ob Ihr Feed erfolgreich verarbeitet wird, besteht darin, das Aussehen Ihrer Assets zu überprüfen, entweder mit der Benutzeroberfläche von Video Cloud Studio oder der Backlot-API /v2/assets anfragen.

Nachdem die Videos und alle anderen zugehörigen Dateien verarbeitet wurden, hosten Sie einfach eine weitere MRSS-Datei am vereinbarten Speicherort, um weitere Dateien abzurufen.
